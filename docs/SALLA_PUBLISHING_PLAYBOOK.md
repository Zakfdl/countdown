# Salla Publishing Playbook

## Pre-Submission Checklist
1. Register Salla developer account.
2. Configure app OAuth redirect URL.
3. Host production backend on HTTPS.
4. Create privacy policy and terms URLs.
5. Prepare app icon, hero image, and screenshots.
6. Verify install/uninstall lifecycle.
7. Verify campaign save + storefront render.
8. Verify billing plan behavior.
9. Prepare Arabic and English listing copy.

## Developer Accounts Needed
- Salla Partner/Developer account
- Hosting platform account
- Domain + SSL provider
- Optional analytics/monitoring accounts

## Submission Assets
- App name + subtitle
- Short and long descriptions
- Feature bullets
- Screenshots (dashboard + storefront)
- Support contact + policy links
- Privacy policy + terms of service URLs

## QA Before Submission
- OAuth flow and token refresh
- Merchant isolation (no cross-merchant data)
- Timer accuracy with timezone differences
- Mobile rendering on product/home pages
- Expiry behavior correctness
- Webhook/event reliability

## Common Rejection Reasons
- Incomplete OAuth flow
- Broken uninstall handling
- Missing legal URLs
- Weak merchant onboarding
- Inaccurate feature claims
- Poor Arabic localization quality

## Improve Approval Chance
- Provide demo merchant account
- Include clear install and setup walkthrough video
- Ensure support response promise is realistic
- Ship polished bilingual UI copy

## Post-Launch Checklist
- Track activation rate (install → first campaign)
- Track weekly active merchants
- Monitor churn by plan
- Run onboarding emails + in-app tips
- Release 1 improvement every 2 weeks
